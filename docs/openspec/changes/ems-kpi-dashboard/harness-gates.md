# EMS 质量门禁指标说明 (Harness Gates)

本文件规定 EMS 系统开发必须通过的质量门禁。除类型检查和构建外，前端必须通过高保真视觉门禁。

---

## 门禁 1: 规格就绪 (Gate 1: Spec Ready)
- **要求**：需求、接口缺口、视觉方向和验收口径均已明确。
- **验证项**：
  1. `proposal.md`、`spec.md`、`plan.md`、`tasks.md`、`api-gap-analysis.md` 已更新。
  2. `frontend-fidelity-spec.md` 已补充，并明确四页视觉、布局和交互细节。
  3. `design-review.md` 已补充，并明确是否同意最新设计、设计边界和开发前确认条件。
  4. 视觉方向明确为四张原型图的浅色工业 KPI 看板，不采用深色 glassmorphism。
  5. `openapi.yaml` 覆盖官方能源接口和新增 mock/扩展接口。
  6. 扩展接口/字段与官方接口边界清晰。
  7. 用户确认文档前不得进入前端编码实现。

---

## 门禁 2: 代码质量 (Gate 2: Code Quality)
- **要求**：代码无编译错误，前端可成功生产构建。
- **验证项**：
  1. `ems-frontend` 运行 `npm run typecheck` 通过。
  2. `ems-frontend` 运行 `npm run build` 通过。
  3. 无未使用导入、明显调试代码、隐式 `any` 或阻塞性构建警告。

---

## 门禁 3: 运行时烟雾测试 (Gate 3: Runtime Smoke Tests)
- **要求**：前端可独立运行，真实能源接口或前端 mock adapter 通讯链路畅通。
- **验证项**：
  1. 前端在 `5173` 启动。
  2. 能源与碳排页面请求路径严格对齐 `/api/kpi/energy/*`。
  3. 真实服务可访问时，前端通过代理调用 `http://10.105.64.36:20003/api`。
  4. 真实服务不可访问或接口缺失时，前端 mock adapter 能返回页面所需数据，页面不崩溃。

---

## 门禁 4: 高保真视觉门禁 (Gate 4: Frontend Fidelity Gate)
- **要求**：四个页面必须与原型高保真对齐，不能只做到功能可用。
- **验证项**：
  1. 使用 1920px 宽桌面浏览器分别打开四页并截图。
  2. 将截图与 `EMS1.png`、`EMS2.png`、`EMS3.png`、`EMS4.png` 对比。
  3. 每页至少核对 8 个项目：布局、背景、顶部导航、筛选条、卡片圆角/边框/阴影、字体层级、图表/画布/表格、交互状态。
  4. 作业效率页额外核对：作业统计大区块、能耗/效率/设备卡片、右侧双图、底部三图布局。
  5. 关系网页额外核对：点阵画布、节点颜色和图标、连线密度、右下角图例入口。
  6. 调度分析页额外核对：两张主曲线、右侧指标卡、底部时间轴色块和刻度密度。
  7. 能源与碳排页额外核对：三组单箱横向堆叠条、两张趋势图、分时电价表、百公里能耗表。
  8. 如出现明显颜色偏差、布局漂移、文本溢出、图表样式不符或原型关键模块缺失，必须修复后才能通过。

---

## 门禁 5: 交互验收 (Gate 5: Interaction Gate)
- **要求**：主要交互具备真实反馈。
- **验证项**：
  1. 顶部四个 Tab 可切换且选中态正确。
  2. 查询/重置可触发数据刷新或状态复位。
  3. 快捷时间、自定义时间、设备下拉、图表筛选下拉均有选中态和刷新反馈。
  4. 图表 Tooltip 和 Legend 可用。
  5. 关系网节点支持悬停/点击反馈，图例可展开/收起。
  6. 调度事件时间轴支持 hover 与点击查看详情，横向滚动不造成定位错位。
  7. 表格行 hover、涨跌箭头、按钮 hover/active/focus 状态完整。
