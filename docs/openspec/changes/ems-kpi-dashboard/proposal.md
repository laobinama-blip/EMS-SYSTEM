# EMS KPI 看板项目建议书 (Proposal)

## 1. 项目背景
本项目旨在基于 4 张高保真原型图开发一套前后端分离的能源管理系统 (EMS) KPI 监控看板。当前正式后端接口文档仅覆盖《能源与碳排》页面，其他页面接口需要通过 Express mock/API adapter 隔离补齐，便于前端开发和后续生产接口替换。

本项目的前端实现必须以原型图为设计源，不允许改成其他视觉风格。后续开发按“高保真复刻业务看板”验收，而不是仅实现功能可用。

---

## 2. 项目范围

### 前端监控看板 (`ems-frontend`)
- **统一布局容器 (`AppShell`)**：实现 Reewell 品牌头部、顶部页签导航、右侧告警胶囊、通知/状态按钮和头像区域。
- **全局筛选器 (`FilterBar`)**：支持 2h、8h、1天、3天、7天、自定义日期范围、设备筛选、查询和重置。
- **高保真视觉实现**：必须保持原型的浅色工业 KPI 看板风格，包括白色顶部栏、浅灰页面背景、白色圆角卡片、细边框、轻阴影、绿色/灰色图表双序列、蓝色查询按钮和红色告警胶囊。
- **四个核心页面**：
  1. **作业效率**：作业统计、能耗、效率、设备统计，以及车辆效率、空驶率、岸桥效率、场桥效率、等待时间等图表。
  2. **关系网**：交互式 SVG 拓扑图，展示 QC、YC、车辆、区块、船舶、充换电站等节点和连线。
  3. **调度分析**：算法效率提升曲线、Hymala 稳定性曲线、指标卡和二次调度事件时间轴。
  4. **能源与碳排**：单箱能耗/成本/碳排、能耗趋势、传统电力 vs 绿电、分时电价、单车百公里能耗表。

### 后端模拟服务 (`ems-backend`)
- 使用 Node.js + Express + TypeScript 搭建 mock/API adapter。
- 能源与碳排页面的官方接口路径必须保持 `/api/kpi/energy/*`，并与《02_第三页_能源与碳排接口与表结构梳理.md》对齐。
- 作业效率、关系网、调度分析等缺失模块使用 `/api/kpi/efficiency/*`、`/api/kpi/network/*`、`/api/kpi/dispatch/*` 等 mock API。
- 使用 Swagger/OpenAPI 提供接口契约，明确哪些字段是官方接口字段，哪些字段是 EMS 看板扩展字段。

---

## 3. 范围外说明
- 暂不连接真实生产 PostgreSQL 或 InfluxDB。
- 暂不运行真实调度算法引擎或 Hymala 模型服务。
- 暂不实现真实 WebSocket 告警推送、认证和权限控制。
- 暂不进行云端生产部署。

---

## 4. 架构决策
- **前端技术栈**：React + TypeScript + Vite + Vanilla CSS。
- **视觉方向**：严格采用原型图的浅色看板风格，禁止使用与原型冲突的深色 glassmorphism 主题。
- **图表库**：Recharts 用于折线图/柱状图，原生 SVG 用于关系网和时间轴。
- **后端技术栈**：Express + TypeScript。使用内存 mock 数据和可替换 service 层模拟 WI 快照表、能耗事实表、车辆时序数据和配置表。

---

## 5. 风险与缓解措施

| 风险点 | 影响程度 | 缓解措施 |
| :--- | :--- | :--- |
| API 缺口 | 高 | 用 mock API 补齐，并在 OpenAPI 中标注官方接口与扩展接口边界。 |
| 视觉偏离原型 | 高 | 将四张原型作为验收源，开发后必须进行浏览器截图对比和差异清单修复。 |
| 高分辨率布局适配 | 中 | 以 1920px 原型为主验收视口，使用 Grid/Flex 保持卡片密度和图表比例。 |
| 关系图交互性能 | 中 | 使用原生 SVG 和局部状态更新，避免引入过重画布框架。 |

---

## 6. 回滚计划
所有开发以 GitHub 仓库 `laobinama-blip/EMS-SYSTEM` 的 `main` 分支为基准。若需要回滚，回退对应提交或删除新增工程目录；文档与代码变更必须可由 Git 历史追溯。

---

## 7. 验收标准
- 顶部导航可正常切换四个页面，无闪烁、错位或布局移位。
- 全局筛选条参数可正确传递并触发数据刷新，重置恢复默认状态。
- 图表 Tooltip、Legend 和节点悬停/点击反馈正常。
- 能源与碳排页面接口路径完全对齐官方能源接口文档。
- 前端视觉必须高保真贴近四张原型图，尤其是顶部栏、筛选条、卡片圆角/边框/阴影、图表配色、关系网节点和时间轴密度。
- `npm run typecheck` 与 `npm run build` 通过。
