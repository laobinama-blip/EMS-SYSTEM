## Git 最新设计方案评估与确认

> [!NOTE]
> **结论：同意 Git 上的最新设计方案，并按 `design-review.md` 的边界执行。**
> 该设计方案在保留浅色工业看板专业视觉的基础上，理清了“高保真视觉还原”与“完整交互流程”的核心目标，且通过前端 fixtures 机制解决了接口未就绪的阻碍风险。当前仍处于文档确认阶段，用户确认前不进入编码实现。

我们对 Git 最新设计的深度分析如下：

1. **视觉规范的克制与严谨**
   - 彻底摒弃大面积渐变与营销型暗色毛玻璃（glassmorphism），回归 `#f4f6fa` 的低噪浅灰工业背景、白色小圆角卡片、`#dfe3ea` 细边框与极轻阴影。这与真实工业场景中日光/强光下的高可读性看板需求一致。
   - 强调双序列对比的专属配色（调度后亮绿 `#19d98f`，调度前中灰 `#858c8c`；操作主色 `#35b9f6`），图表视觉聚焦且含义明确。
2. **核心交互流的完整覆盖**
   - **顶栏导航**：`AppShell` 与 `FilterBar` 整体联动，页签切换响应瞬时，告警气泡常驻显示。
   - **全局筛选**：快速时间切换（2h/8h/1天等分段控制）、自定义时间长胶囊、设备多选、查询/重置动画反馈。
   - **拓扑网 (Page 2)**：基于 SVG Canvas，不依赖笨重的第三方库，可完全控制节点悬停、连线高亮、图例折叠以及节点详细信息浮层的展开交互。
   - **调度分析甘特图 (Page 3)**：二次调度横向时间轴带滚动条，工况胶囊块按时间精确定位，悬停显示冲突原因，点击展示解决方案，完全满足工业调度过程回溯的闭环。
   - **能源分时与百公里 (Page 4)**：三段单箱能耗多段堆叠条（柴油/火力/绿电百分比），分时避峰表（42%）与百公里能耗列表（红绿趋势箭头），能高度还原数据趋势。

## 4 个原型页面的视觉布局分析与核对点 (Fidelity Analysis)

我们将逐页严格实现以下模块结构：

### 1. 顶部全局外壳 (`AppShell` & `FilterBar`)
- **`AppShell` 顶栏**：
  - 左侧：展示 `Reewell World | KPI`，位置、大小和原型一致。
  - 中间：横向页签导航（作业效率、关系网、调度分析、能源与碳排），选中状态为文字加黑并在底部呈现黑色粗线。
  - 右侧：红色告警胶囊（带左侧告警小图标）、通知气泡、设置齿轮及用户头像。
- **`FilterBar` 筛选条**：
  - 包裹在一个长条形的纯白圆角容器内。
  - 时间快速切换：分段按钮组（2h、8h、1天、3天、7天），选中项呈现白色圆角胶囊状，并带有轻微阴影；非选中态为浅灰底。
  - 日期选择：包含开始时间与结束时间输入框，并在中间以“-”连接。
  - 设备多选：支持按设备类型和编码筛选。
  - 动作按钮：亮蓝色背景的“查询”按钮，以及灰色描边的“重置”按钮。

### 2. 作业效率页 (`EMS1.png` - 1920x1733)
- **顶部作业统计区**：
  - 左侧为“全部作业统计”大分区，右侧为装船、卸船、转堆、未知任务四个分类分区，形成左 1 + 右 2x2 的结构。
  - 每组显示 TEU/自然箱/车辆循环等核心指标，并展示 20尺、40尺、45尺箱量细分。
  - 分区之间使用细分割线，不额外嵌套卡片。
- **中部统计与右侧图表区**：
  - 左侧依次为“能耗”“效率”“设备”三组白色统计卡。
  - 右侧竖向排列“车辆平均效率”和“车辆平均空驶率”两张双折线图。
- **底部图表区**：
  - 三列展示“岸桥效率”“场桥效率”“车辆平均等待时间”，前两者为双折线图，等待时间为双柱状图。
  - **色彩要求**：调度后使用亮绿色，调度前使用中性灰色。

### 3. 关系网拓扑页 (`EMS2.png` - 1920x1080)
- **画布背景**：浅灰色底色，叠加均匀的点阵网格线 (Grid dots pattern)。
- **拓扑节点**：手写渲染 SVG 元素，准确表达岸桥 (QC)、场桥 (YC/RTG)、集卡车辆 (Vehicle)、箱区 (Block)、船舶 (Ship)、充换电站。
- **拓扑连线**：实线与虚线，体现绑定或当前任务指派关系，任务活跃状态可采用动画流光或颜色标记。
- **图例与浮层**：右下角固定浮动图例面板，可折叠，显示各种状态和节点图标；点击节点时在侧边或节点旁弹出卡片详情。

### 4. 调度分析页 (`EMS3.png` - 1920x1080)
- **图表侧**（占主宽度）：
  - “算法效率提升”对比曲线（Recharts AreaChart / LineChart，多算法对比）。
  - “Hymala 世界模型稳定性”时序图（单灰色曲线，Y轴为延迟，含指标）。
- **指标卡侧**（右侧）：
  - 算法健康度汇总指标（成功率、超时次数、总调用次数）。
- **底端二次调度甘特时间轴**：
  - 横向时间轴刻度。
  - 任务事件块（如“耳轴换介”、“重选退出”、“等待复位”），呈现圆角胶囊块，颜色需与原型一致，点击后可展示冲突原因。

### 5. 能源与碳排页 (`EMS4.png` - 1920x1329)
- **顶栏统计条**：
  - **单箱能耗**：水平多段堆叠条（QC、水平运输、YC、外集卡），各段以柴油、火力发电、绿电的百分比构成。
  - **单箱综合成本**：根据单箱能耗折算的 RMB/TEU 金额结构。
  - **单箱碳排放量**：根据单箱能耗折算的 tCO2/TEU 分布结构。
- **中部趋势区**：
  - **总能耗趋势**：调度前与调度后能耗对比折线图.
  - **传统电力 vs 绿电**：趋势线与绿色能源占比折线图。
- **底部详细对照**：
  - **分时电价避峰率**：左侧显示避峰指标（42%），右侧展示 6 个分时时段的用电量及电费比对。
  - **单车百公里能耗表**：展示车辆编码，以及调度前/调度后的百公里能耗 (kWh/100km)、成本 (RMB/100km)、碳排对照列表。

## 前端高保真实现补充规范 (Supplements)

为保证在编码实现阶段万无一失，我们将补充执行以下四项具体规范：

1. **自绘 SVG 精准资产替代**
   - 针对船舶、岸桥、场桥、集卡、充电桩和换电站图标，Lucide 图标库如果语义不符，我们将编写内联 SVG (Inline SVG) 路径，线宽统一使用 `1.5px` 或 `2px`，确保在 Page 2 关系网画布及卡片标题中视觉风格整齐划一。
2. **拓扑画布固定坐标阵列 (Grid Coordinate Matrix)**
   - 关系网（Page 2）节点的位置分布，我们将严格根据 1920x1080 分辨率中原型图的相对比例换算为固定像素坐标 `(x, y)`（如船舶 `x: 120, y: 350`，岸桥 `x: 350` 等）。禁止使用动态受力导向布局（Force-directed layout）以避免连线缠绕或布局漂移。
3. **甘特时间轴与彩色事件胶囊定位**
   - 调度分析（Page 3）的二次调度甘特图，时间轴起点与终点需与事件 `startEpoch` 精确计算并转换为 `left` 与 `width` 百分比。胶囊圆角设为 `16px`，悬停时触发 tooltip 解释“冲突原因”，点击时在中心或右侧划出“冲突解决方案浮层”。
4. **分时电价区间与避峰率一致性**
   - 能源页（Page 4）的分时电价表格需与原型中的 6 个时段完全一致（如 `00:00-02:00` 谷期，`08:00-12:00` 峰期等）。表格中调度前后的电量电费变化需严格遵守 `fixtures.ts` 中定义的数值，确保最终计算出的避峰率（42%）与原型图完美吻合。
5. **长页面截图策略**
   - Page 1 和 Page 4 的原型高度超过 1080px，验收时必须使用全页截图或分段截图；不得只截首屏后宣称通过。
   - Page 2 和 Page 3 接近单屏画布，可使用 1920x1080 视口验收，但仍需检查筛选条、画布边距和底部控件是否完整。
