# EMS KPI 看板技术规格说明书 (Specification)

## 需求 1: 应用程序架构
系统 MUST 以 React 前端与 Node.js Express 后端解耦的前后端分离架构运行。

### 验收标准
- **Given** 后端 mock/API adapter 已启动；**When** 前端加载页面或点击查询；**Then** 所有统计数据与图表数据必须通过 HTTP 请求拉取，请求前缀为 `/api/kpi/*`。
- **Given** `AppShell` 已渲染；**When** 用户点击顶部导航页签；**Then** 主内容区切换到对应页面，导航选中态同步更新。

---

## 需求 2: 全局筛选与重置
系统 MUST 提供统一 `FilterBar`，联动控制四个页面的查询范围。

### 验收标准
- **Given** 筛选器加载完成；**When** 用户选择 2h、8h、1天、3天、7天或自定义时间并点击“查询”；**Then** 当前页面重新请求 KPI 数据并保持筛选选中态。
- **Given** 用户已应用非默认筛选；**When** 点击“重置”；**Then** 时间回到 `1天`，日期输入清空，设备回到 `ALL`，页面以默认参数重载。

---

## 需求 3: 高保真前端设计
系统 MUST 以四张原型图为设计源进行高保真实现，不得改变为深色主题或其他设计风格。

### 验收标准
- **Given** 前端在 1920px 宽桌面视口打开；**When** 逐页对比 `EMS1.png`、`EMS2.png`、`EMS3.png`、`EMS4.png`；**Then** 页面结构、浅色背景、顶部导航、筛选条、卡片布局、图表区域、表格密度和关系图画布必须与原型高度一致。
- **Given** 页面组件渲染完成；**When** 检查视觉细节；**Then** 卡片圆角、边框、阴影、字体层级、图标尺寸、绿色/灰色图表双序列、蓝色查询按钮、红色告警胶囊必须贴近原型。
- **Given** 浏览器截图与原型存在差异；**When** 差异属于布局漂移、颜色明显不符、文本溢出、卡片比例失真或图表样式不符；**Then** 必须修复后才能通过视觉门禁。

---

## 需求 4: 作业效率页
系统 MUST 呈现作业吞吐指标、能耗/效率/设备统计和 5 组调度效果图表。

### 验收标准
- **Given** 用户进入“作业效率”；**When** 页面加载完成；**Then** 显示全部作业、装船、卸船、转堆、未知任务等统计区域，并展示 TEU、自然箱、车辆循环和 20/40/45 尺箱量。
- **Given** 后端返回双序列数据；**When** 图表渲染；**Then** 车辆平均效率、空驶率、岸桥效率、场桥效率、平均等待时间均展示调度前/调度后对比。

---

## 需求 5: 关系网拓扑页
系统 MUST 实现展示调度资源链路的 SVG 拓扑关系图。

### 验收标准
- **Given** 用户进入“关系网”；**When** 组件载入；**Then** 画布渲染 QC、YC/RTG、Vehicle、Block、Ship、Charging/Swapping Station 等节点和连线。
- **Given** 节点处于正常状态；**When** 用户悬停或点击节点；**Then** 节点放大/高亮并展示节点信息。
- **Given** 关系图在 1920px 视口展示；**When** 对比第二张原型；**Then** 点阵背景、节点颜色、连线疏密、右下角图例入口应接近原型。

---

## 需求 6: 调度分析页
系统 MUST 展示调度算法优化成效、Hymala 稳定性和二次调度时间轴。

### 验收标准
- **Given** 用户进入“调度分析”；**When** 页面初始化；**Then** 显示算法效率提升对比曲线、Hymala 稳定性曲线、指标卡和二次调度事件时间轴。
- **Given** 时间轴渲染完成；**When** 事件按时间分布；**Then** 每个事件块应正确定位、保持原型中的色彩胶囊样式，并支持点击查看详情。

---

## 需求 7: 能源与碳排页
系统 MUST 与官方能源接口路径对齐，并呈现原型中的能耗结构、趋势、分时电价和百公里分析。

### 验收标准
- **Given** 后端暴露 `/api/kpi/energy/single-box-energy-statistics`；**When** 能源页加载；**Then** 单箱能耗组件渲染 QC、水平运输、YC、外集卡四段结构，并展示柴油、火力、绿电占比。
- **Given** 后端返回 `/api/kpi/energy/time-of-use-electricity`；**When** 分时电价组件加载；**Then** 指标区显示调度前电费、调度后电费、节省金额、平均电价、避峰率，并展示 6 个分时时段。
- **Given** 后端返回 `/api/kpi/energy/energy-trend`；**When** 总能耗趋势图加载；**Then** 图表显示调度前与调度后两条折线。
- **Given** 后端返回 `/api/kpi/energy/vehicle-energy-per-100km`；**When** 百公里能耗表渲染；**Then** 表格逐行展示每辆车调度前/后的能耗、成本和碳排。

---

## 需求 8: 质量门禁与验证
项目 MUST 包含可重复执行的类型检查、构建和视觉验收流程。

### 验收标准
- **Given** 执行类型检查；**When** 运行 `npm run typecheck`；**Then** 前后端 TypeScript 检查通过。
- **Given** 执行生产构建；**When** 在前端目录运行 `npm run build`；**Then** Vite 构建成功。
- **Given** 完成前端实现；**When** 进行浏览器截图对比；**Then** 输出原型对比结果，未修复的高保真差异不得进入最终交付。
