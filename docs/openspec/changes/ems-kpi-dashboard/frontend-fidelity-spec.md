# EMS KPI 前端高保真与交互规范

本文件是前端开发的最高优先级设计约束。当前阶段先不以接口为核心，所有页面数据可由本地 fixtures 驱动；开发目标是把 `EMS1.png`、`EMS2.png`、`EMS3.png`、`EMS4.png` 中的视觉、布局、密度和交互流程高保真还原。

---

## 1. 总体原则

- **原型优先**：四张 PNG 是视觉事实源，OpenAPI 与 mock 数据只服务于页面还原。
- **浅色工业看板**：禁止深色主题、glassmorphism、大面积渐变、营销页式 hero 或装饰性背景。
- **像素级纪律**：页面背景、顶部栏、筛选条、卡片圆角、边框、阴影、字体层级、图表颜色、表格密度必须以 1920px 原型为基准。
- **完整交互**：Tab、筛选、查询、重置、下拉、图表 tooltip、拓扑节点、图例、时间轴事件、表格悬停都必须有真实反馈。
- **无接口阻塞**：接口不可用时不得影响视觉与交互验收；前端 fixtures 必须能完整驱动四页。

## 2. 全局视觉系统

| 项目 | 规范 |
| :--- | :--- |
| 页面背景 | `#f1f3f7` / `#f4f6fa`，浅灰、低噪声、无装饰图案 |

## 3. 全局 AppShell

### 顶部导航

- 左侧展示 `Reewell World | KPI`，位置、大小和原型一致。
- 中央导航包含四个页签：`作业效率`、`关系网`、`调度分析`、`能源与碳排`。
- 当前页签使用黑色粗底线，图标和文字颜色加深；非当前页签为黑色/深灰普通态。
- 点击页签必须立即切换页面，筛选条保持在页面顶部。

### 右侧状态区

- 告警胶囊文案：`QC301效率 25.4箱/h，低于阈值30箱/h`。
- 工具按钮为圆形白底，有阴影和浅边框；悬停时轻微上浮或边框加深。
- 头像保持圆形外框，图像可用本地静态渐变/图片 fixture，但尺寸必须贴近原型。

### 全局筛选条

- 筛选条为白色大圆角容器，位于顶栏下方，宽度与主体内容对齐。
- 快捷时间为分段控件：`2h`、`8h`、`1天`、`3 天`、`7 天`、`自定义`。
- 默认选中 `1天`，选中态白色胶囊加轻阴影。
- 日期输入区域为浅灰长胶囊，展示 `开始时间 ~ 结束时间`。
- 查询按钮为蓝色胶囊，重置按钮白底灰边。
- 点击查询时页面出现短暂加载态或数据刷新反馈；点击重置恢复默认筛选。

## 4. 页面一：作业效率

### 布局结构

- 第一屏主体为两列布局：左侧宽区域展示作业、能耗、效率、设备，右侧窄列展示两张趋势图。
- 顶部作业统计大卡片横跨左侧主区域，内部左边为“全部作业统计”，右边为 2x2 分类统计。
- 下方依次为能耗横向指标条、效率横向指标条、设备统计大卡片。
- 页面底部为三列图表：岸桥效率、场桥效率、车辆平均等待时间。

### 必须还原的组件

- **全部作业统计**：总作业箱量、自然箱量、车辆循环、20/40/45 尺箱，底部四个分类小块。
- **分类作业统计**：装船、卸船、转堆、未知任务；每块包含 TEU、自然箱、车辆循环、20/40/45 尺箱。
- **能耗卡片**：总能源投入金额、调度节能成本、燃油总消耗、碳排、耗电总计、电力负荷峰值、平均每箱耗电。
- **效率卡片**：全场平均效率、岸桥平均效率、场桥平均效率、泊位效率、平均在泊时间。
- **设备卡片**：全部设备统计、单车作业统计、单车平均效率三段，保留分割线和指标密度。
- **图表**：车辆平均效率、车辆平均空驶率、岸桥效率、场桥效率为双折线；车辆平均等待时间为双柱状。

### 交互要求

- 图表 hover 显示白色 tooltip，包含时间点、调度前/后值和相关箱型拆分。
- 图表上方下拉 `全部车辆` / `全部岸桥` / `全部场桥` 可切换选中态并刷新本地数据。
- 查询/重置必须刷新当前页所有统计卡和图表。

## 5. 页面二：关系网

### 布局结构

- 页面主体是一个大画布卡片，几乎占满视口剩余空间。
- 背景为细密点阵，点阵颜色极浅，不可喧宾夺主。
- 拓扑图节点分散在画布中，连线为浅灰实线/虚线，保留原型中的疏密节奏。
- 右下角有 `图例` 入口和下拉箭头。

### 节点规范

- 车辆节点：绿色圆形，白色车辆图标，标签如 `T001`、`T004`。
- QC 节点：蓝色圆形，白色岸桥/吊具图标，标签如 `QC301`。
- RTG/YC 节点：紫色圆形，白色场桥图标，标签如 `RTG01`、`RTG02`。
- Block 节点：绿色方形/圆角块，白色堆场图标，标签如 `Block 1`。
- 船舶节点：蓝色节点，标签 `船舶`。
- 充电桩：黄色圆形，闪电图标，标签 `充电桩 01`。
- 换电站：青绿色圆形，换电图标，标签 `换电站`。

### 交互要求

- 节点 hover：节点轻微放大，连线高亮，显示简洁 tooltip。
- 节点点击：保持选中态，相关连线变深，右下或节点附近显示节点信息浮层。
- 画布支持轻量拖拽或至少支持节点点击反馈；不得出现页面跳动。
- 图例点击展开/收起，显示节点类型和颜色说明。

## 6. 页面三：调度分析

### 布局结构

- 第一张大卡片：左侧“算法效率提升对比”折线图，右侧 2x2 算法统计卡。
- 第二张大卡片：左侧“Hymala 世界模型稳定性波动曲线”，右侧 2x2 服务指标卡。
- 第三张大卡片：二次调度分析横向时间轴，底部带横向滚动条。

### 必须还原的组件

- 算法效率提升图：灰色未使用算法预测结果、绿色 Hymala 优化调度后，绿色面积浅填充。
- 算法统计卡：算法统计、Hymala、OR、混合；包含调用次数、效率提升，正向绿色、负向红色。
- 稳定性曲线：单灰色曲线，Y 轴响应时间 ms。
- 服务指标卡：接口调用成功率、服务响应时延、接口调用总次数、服务超时率。
- 时间轴：顶部时间刻度、左侧 `工况` 行头、彩色胶囊事件块、当前时间竖线。

### 交互要求

- 两张曲线 hover 显示 tooltip。
- 指标卡 hover 有轻微背景变化。
- 时间轴事件 hover 显示事件详情；点击后保持选中态。
- 时间轴横向滚动条可拖动，事件定位不应错位。

## 7. 页面四：能源与碳排

### 布局结构

- 两列布局：左侧为三组单箱结构条 + 分时电价表，右侧为两张趋势图 + 单车百公里能耗表。
- 左侧上卡片包含三段：单箱能耗、单箱综合成本、单箱综合碳排，每段都有标题、右侧总值、横向分段条和三行能源占比。
- 右侧两张趋势图上下排列，底部右侧为表格。

### 必须还原的组件

- **单箱结构条**：QC、水平运输、YC、外集卡四段，颜色为浅蓝、薄荷绿、淡紫、浅黄。
- **能耗趋势**：调度前/调度后双折线，右上设备下拉。
- **传统电力 vs 绿电**：双折线，右上绿色占比胶囊。
- **分时电价**：顶部 5 个指标，下面按 `00:00-02:00`、`02:00-08:00`、`08:00-12:00`、`12:00-18:00`、`18:00-22:00`、`22:00-24:00` 展示调度前后电量与电费。
- **百公里能耗表**：设备列 + 能耗/成本/碳排三组，每组包含调度前/调度后。

### 交互要求

- 设备下拉切换后刷新两张趋势图和百公里表。
- 表格行 hover 背景变浅灰，调度后下降值显示绿色箭头，上升值显示红色箭头。
- 三组单箱结构条 hover 显示对应环节名称和值。

## 8. 响应式与边界

- 1920px 是第一验收视口，必须优先保证与原型一致。
- 1440px 允许等比压缩列宽，但不得造成文字溢出、图表重叠或表格破版。
- 移动端不是首要设计源，但页面应可纵向浏览，顶部导航可横向滚动，卡片改为单列。
- 所有按钮、下拉、tooltip、事件块的文本不得溢出容器。

---

## 9. 验收流程

每次前端开发完成后必须执行：

1. 在 1920px 宽桌面视口打开四页。
2. 分别截图并与四张原型图对照；Page 1 与 Page 4 必须使用全页截图或分段截图，Page 2 与 Page 3 可用 1920x1080 单屏截图。
3. 每页至少记录 8 个核对点：布局、背景、顶部栏、筛选条、卡片、字体、图表/画布/表格、交互状态。
4. 手动点击完整流程：页签切换、筛选查询、重置、下拉、图表 hover、拓扑节点 hover/click、图例展开、时间轴事件 hover/click、表格 hover。
5. 若出现明显视觉偏差或交互缺失，不得进入交付或提交代码。

---

## 10. 开发前置条件

在用户确认本文档前，不进入前端编码实现。确认后，开发顺序为：

1. 搭建 `ems-frontend`。
2. 建立 design tokens 与基础组件。
3. 用 fixtures 完整驱动四页。
4. 逐页实现并截图对比。
5. 修复差异直到达到高保真门禁。
