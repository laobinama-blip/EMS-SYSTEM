# EMS KPI 最新设计评审结论

## 1. 评审结论

**同意最新设计方向，但需要按本文的约束执行。**

四张原型图已经形成清晰、统一、可落地的浅色工业 KPI 看板设计体系。当前不需要重新生成新视觉方案，也不建议改成深色、毛玻璃、营销化或装饰性更强的风格。后续开发应以 `EMS1.png`、`EMS2.png`、`EMS3.png`、`EMS4.png` 为唯一视觉事实源，先保证前端高保真还原，再处理真实接口联调。

本次评审同时确认：**当前仓库仍处于文档确认阶段，不进入前端编码实现。**

---

## 2. 同意的设计点

### 2.1 视觉方向正确

- 浅灰背景 + 白色卡片 + 细边框 + 轻阴影适合工业运营看板，信息密度高但不压迫。
- 顶部导航、告警胶囊、筛选条和内容卡片的层级清楚，符合重复使用的调度监控场景。
- 调度前灰色、调度后绿色的双序列表达明确，适合所有趋势图复用。
- 蓝色查询按钮、红色告警、绿色收益指标的语义颜色清晰，不需要额外解释性文案。

### 2.2 页面结构完整

- **作业效率**：覆盖作业量、能耗、效率、设备和多组调度效果图，适合作为默认首页。
- **关系网**：大画布拓扑图符合调度资源链路展示方式，节点、连线、图例都能自然扩展交互。
- **调度分析**：算法效率、模型稳定性和二次调度时间轴构成完整分析闭环。
- **能源与碳排**：单箱能耗、成本、碳排、趋势、分时电价和百公里表格覆盖能源分析主路径。

### 2.3 前端优先策略正确

- 当前接口文档不能完整覆盖四页展示数据，因此先用前端 fixtures 完整驱动页面是合理策略。
- 接口联调不应阻塞高保真验收；真实接口后续只替换数据源，不改变页面结构。

---

## 3. 必须坚持的设计约束

### 3.1 不重新设计

开发时不得把页面改成：

- 深色大屏。
- glassmorphism 毛玻璃。
- 大面积渐变背景。
- 营销型 hero 或宣传页结构。
- 卡片堆叠式通用后台模板。

### 3.2 不“功能先行”牺牲细节

以下细节必须优先还原：

- 顶部品牌、导航位置、选中底线、右侧告警胶囊。
- 筛选条的大圆角容器、分段按钮、日期输入、查询/重置按钮。
- 卡片圆角、边框、阴影、分割线和内部留白。
- 图表虚线网格、legend、tooltip、线条圆点和灰/绿配色。
- 关系网点阵背景、节点颜色、节点图标、标签位置和连线疏密。
- 调度时间轴刻度、当前时间竖线、彩色事件胶囊和底部滚动条。
- 能源页堆叠条、分时电价表和百公里能耗表格密度。

### 3.3 100% 高保真的解释

这里的“100% 还原”指产品验收口径上的高保真一致：

- 1920px 主视口下布局、比例、颜色、密度、组件状态必须接近原型。
- 没有 Figma 源文件、精确字体文件、品牌 logo 矢量源、头像原图和完整图标库时，允许采用视觉等价实现。
- 任何视觉等价替代都必须服务于原型一致性，不能引入新的视觉语言。

## 4. 当前仍需注意的风险

| 风险 | 说明 | 处理方式 |
| :--- | :--- | :--- |
| 无 Figma 标注 | 原型只有 PNG，缺少精确间距、字号、颜色 token | 以截图量测 + 浏览器截图对比微调 |
| 品牌与头像资产缺失 | `Reewell World` 标志和头像可能无法 1:1 复刻 | 优先使用文字/近似资产，占位尺寸必须一致 |
| 图标源不明确 | 顶栏、节点、卡片标题图标缺少明确 icon set | 使用 lucide 或自绘 SVG，保持线宽、大小和语义一致 |
| 图表数据非真实 | 当前先不管接口，数据由 fixtures 驱动 | fixtures 必须覆盖 tooltip、筛选、hover、选中等状态 |
| 关系网坐标复杂 | 拓扑图节点多，连线密度和位置是高风险点 | 固定坐标布局优先，后续再考虑自动布局 |
| 任务状态误导 | 远端曾出现“前端骨架已完成”的任务描述，但仓库没有代码目录 | 文档任务状态必须改回待开发，避免误判 |

---

## 5. 是否同意最新设计

**同意。**

理由如下：

1. 设计风格与工业 KPI 看板场景匹配，克制、清晰、专业。
2. 四页信息架构完整，覆盖作业效率、资源关系、调度分析、能源碳排四条主线。
3. 高保真实现可行，技术上可以用 React + Vite + CSS + SVG + Recharts 实现。
4. 前端 fixtures 优先的策略能解除接口缺口对页面还原的阻塞。
5. 只要严格执行视觉门禁和交互门禁，方案可以进入前端开发阶段。

---

## 6. 进入开发前的确认条件

在开始前端开发前，需要满足：

1. 用户确认 `frontend-fidelity-spec.md` 和本文 `design-review.md`。
2. 确认当前阶段只做前端，不开发生产后端。
3. 确认接口不阻塞页面还原，fixtures 可以作为首版数据源。
4. 确认开发验收以四张 PNG 的浏览器截图对比为准。

未满足以上条件前，继续停留在文档阶段，不创建或提交前端代码。
