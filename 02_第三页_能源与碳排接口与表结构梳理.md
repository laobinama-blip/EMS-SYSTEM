# 第三页《能源与碳排》接口与依赖表梳理

## 一、IP端口号
- 服务地址：`http://10.105.64.36:20003`
  - IP：`10.105.64.36`
  - 端口：`20003`

## 二、接口清单

| 接口 | 方法 | 作用 | 主要依赖 |
| --- | --- | --- | --- |
| `/api/kpi/energy/device-types` | GET | 设备类型下拉 | 无（静态枚举） |
| `/api/kpi/energy/devices` | GET | 设备编码下拉 | `kpi_work_instruction_snapshot` |
| `/api/kpi/energy/single-box-energy-statistics` | GET | 单箱能耗结构 | `kpi_work_instruction_energy_fact`，回退 `kpi_work_instruction_snapshot` |

| `/api/kpi/energy/energy-trend` | GET | 总能耗趋势 | `kpi_work_instruction_energy_fact` |
| `/api/kpi/energy/energy-ranking` | GET | 设备能耗排名 | `kpi_work_instruction_energy_fact` |

## 三、接口逐项说明

### 1. `/api/kpi/energy/device-types`
- 入参：无
- 返回：固定三类设备
  - `VEHICLE`：集卡 / 水平运输车辆
  - `QC`：岸桥设备
  - `YC`：场桥设备
- 返回字段说明：

| 字段 | 说明 |
| --- | --- |
| `code` | 设备类型编码，前端筛选和后续接口传参使用 |
| `name` | 设备类型展示名称 |

- 说明：纯代码静态返回，不查库。

### 2. `/api/kpi/energy/devices`
- 入参：`deviceTypeCode`

| 字段 | 说明 |
| --- | --- |
| `deviceTypeCode` | 设备类型编码，可选值：`VEHICLE`、`QC`、`YC` |

- 返回字段说明：

| 字段 | 说明 |
| --- | --- |
| `deviceCode` | 设备编码，用于趋势、排名、百公里能耗等接口过滤 |
| `deviceName` | 设备展示名称；无独立名称时通常与设备编码一致 |

- 逻辑：
  - `VEHICLE`：从 `kpi_work_instruction_snapshot.vehicle_no` 去重
  - `QC`：按 `operation_type` 解析桥机编码
  - `YC`：按 `operation_type` 解析场桥编码
- 用途：第三页筛选器的设备下拉。

### 3. `/api/kpi/energy/single-box-energy-statistics`
- 入参：`startTime`、`endTime`

| 字段 | 说明 |
| --- | --- |
| `startTime` | 查询开始时间，格式 `yyyy-MM-dd HH:mm:ss`，闭区间 |
| `endTime` | 查询结束时间，格式 `yyyy-MM-dd HH:mm:ss`，闭区间 |

- 作用：输出单箱能耗结构，分为：
  - `qc`
  - `horizontalTransport`
  - `yc`
  - `externalTruck`（当前固定空数组）
- 核心逻辑：
  1. 优先读取 `kpi_work_instruction_energy_fact`
  2. 若该表有数据，则按 TEU、桥机工时、车辆里程折算单箱能耗
  3. 若事实表为空，则回退到 `kpi_work_instruction_snapshot` 现场重算
- 返回字段说明：

| 字段 | 说明 |
| --- | --- |
| `qc` | 岸桥单箱能耗列表，按能源类型拆分 |
| `horizontalTransport` | 水平运输单箱能耗列表，主要对应车辆运输能耗 |
| `yc` | 场桥单箱能耗列表，按能源类型拆分 |
| `externalTruck` | 外集卡单箱能耗列表，目前固定为空数组 |
| `energyType` | 能源类型，如电、柴油、绿电等，来自能耗画像配置 |
| `energyValue` | 单箱能耗值，按 TEU 折算后的单位箱能耗 |
| `energyUnit` | 能耗单位，通常随能源类型展示，如 kWh、L |

- 依赖配置：`kpi_energy_profile_config`

### 4. `/api/kpi/energy/single-box-energy-cost`
- 入参：`startTime`、`endTime`

| 字段 | 说明 |
| --- | --- |
| `startTime` | 查询开始时间，格式 `yyyy-MM-dd HH:mm:ss`，闭区间 |
| `endTime` | 查询结束时间，格式 `yyyy-MM-dd HH:mm:ss`，闭区间 |

- 作用：输出单箱能耗成本结构。
- 公式：`energyAmount = energyValue * unit_price`
- 返回字段说明：

| 字段 | 说明 |
| --- | --- |
| `qc` | 岸桥单箱能耗成本列表 |
| `horizontalTransport` | 水平运输单箱能耗成本列表 |
| `yc` | 场桥单箱能耗成本列表 |
| `externalTruck` | 外集卡单箱能耗成本列表，目前固定为空数组 |
| `energyType` | 能源类型，用于匹配单价配置 |
| `energyValue` | 单箱能耗值，作为成本计算基数 |
| `unitPrice` | 能源单价，来自 `kpi_energy_price_config` |
| `energyAmount` | 单箱能耗成本金额 |

- 依赖配置：
  - `kpi_energy_profile_config`
  - `kpi_energy_price_config`

### 5. `/api/kpi/energy/carbon-emission-distribution`
- 入参：`startTime`、`endTime`

| 字段 | 说明 |
| --- | --- |
| `startTime` | 查询开始时间，格式 `yyyy-MM-dd HH:mm:ss`，闭区间 |
| `endTime` | 查询结束时间，格式 `yyyy-MM-dd HH:mm:ss`，闭区间 |

- 作用：输出单箱碳排结构。
- 公式：`carbonEmission = energyValue * carbon_factor`
- 返回字段说明：

| 字段 | 说明 |
| --- | --- |
| `qc` | 岸桥单箱碳排列表 |
| `horizontalTransport` | 水平运输单箱碳排列表 |
| `yc` | 场桥单箱碳排列表 |
| `externalTruck` | 外集卡单箱碳排列表，目前固定为空数组 |
| `energyType` | 能源类型，用于匹配碳排系数 |
| `energyValue` | 单箱能耗值，作为碳排计算基数 |
| `carbonFactor` | 单位能源碳排系数，来自 `kpi_energy_carbon_factor_config` |
| `carbonEmission` | 单箱碳排放量 |

- 依赖配置：
  - `kpi_energy_profile_config`
  - `kpi_energy_carbon_factor_config`

### 6. `/api/kpi/energy/energy-trend`
- 入参：

| 字段 | 说明 |
| --- | --- |
| `startTime` | 查询开始时间，格式 `yyyy-MM-dd HH:mm:ss` |
| `endTime` | 查询结束时间，格式 `yyyy-MM-dd HH:mm:ss` |
| `timeGranularity` | 时间粒度，可选 `day` 或 `hour` |
| `deviceCode` | 设备编码，可选；为空时统计全部设备 |

- 返回字段说明：

| 字段 | 说明 |
| --- | --- |
| `statTime` | 统计时间点，按 `timeGranularity` 对齐到天或小时 |
| `totalEnergyConsumption` | 当前时间桶内总能耗 |
| `averageEnergyConsumption` | 当前时间桶内平均能耗，按桶内小时数折算 |
- 计算口径：
  - 先按天/小时分 bucket
  - 每个 bucket 汇总总能耗
  - `averageEnergyConsumption = totalEnergyConsumption / bucket.hours`

### 7. `/api/kpi/energy/energy-ranking`

### 10. `/api/kpi/energy/vehicle-energy-per-100km`
- 入参：

| 字段 | 说明 |
| --- | --- |
| `startTime` | 查询开始时间，格式 `yyyy-MM-dd HH:mm:ss` |
| `endTime` | 查询结束时间，格式 `yyyy-MM-dd HH:mm:ss` |
| `deviceCode` | 车辆编码，可选；为空时统计全部车辆 |

- 返回字段说明：

| 字段 | 说明 |
| --- | --- |
| `deviceCode` | 车辆编码 |
| `energyPer100km` | 百公里能耗，单位通常为 kWh/100km |
| `cost` | 百公里能耗成本，按电价配置折算 |
| `carbonEmission` | 百公里碳排放量，按碳排系数折算 |

#### 10.1 仿真模式 `SIMULATION_ESTIMATE`
- 数据来源：`kpi_vehicle_hourly_metric`
- 关键字段：
  - `mileage_km`
  - `idle_stopped_hours`
  - `low_speed_hours`
  - `start_stop_count`
- 逻辑：
  - 基础百公里能耗 = `simulationDropPercentPerKm * batteryCapacityKwh`
  - 行为修正 = 怠速修正 + 低速修正 + 起停修正
  - 最终百公里能耗 = 基础值 × `(1 + 行为修正)`

#### 10.2 生产模式 `PRODUCTION_BATTERY`
- 数据来源：Influx measurement `fms_vehicle_chassis`
- 关键字段：
  - `vehicleMilesTraveled`
  - `batteryAvailable`
- 逻辑：
  - 先求里程增量
  - 再累加电量下降值
  - `energyPer100km = totalDischargeEnergyKwh / mileageDelta * 100`

| 字段 | 说明 |
| --- | --- |
| `id` | 快照主键 |
| `wi_ref_no` | WI 业务号 |
| `operation_type` | 作业类型，用于区分 QC / YC |
| `vehicle_no` | 车辆号 |
| `container_length` | 箱长，折算 TEU |
| `receive_machine_no` | 接箱设备 |
| `deliver_machine_no` | 交箱设备 |
| `receive_dispatch_time` | 派车时间 |
| `deliver_finish_time` | 完成时间 |
| `recv_start_time` | 接箱开始 |
| `recv_end_time` | 接箱结束 |
| `dlvr_start_time` | 交箱开始 |
| `dlvr_end_time` | 交箱结束 |

## 六、补充说明
- 第三页大部分接口都会先查 `kpi_work_instruction_energy_fact`；只有事实表为空时才会回退实时重算。
- `externalTruck` 目前固定返回空数组，说明外集卡口径尚未落地。
- `device-energy-ratio` 已改成 POST，请求体支持多选设备类型。
- 百公里能耗接口的成本、碳排不来自表，而来自运行配置：
  - `electricityPricePerKwh`
  - `carbonFactorPerKwh`
  - `batteryCapacityKwh`
